# WetoCode 架构决策

## 为什么不重写 Agent Loop

OpenCode 已提供成熟的模型适配、工具调用、会话、权限、MCP、LSP 和上下文压缩能力。WetoCode 的差异化应集中在符合中国开发者习惯的中文体验、常用模型接入、企业安全与治理，而不是重新实现文件编辑循环。

桌面版使用持久 `opencode serve` 和官方 SDK。主进程订阅增量事件、处理权限问答并保存运行状态；CLI 继续复用 OpenCode TUI。这样既保留成熟 Agent Loop，也支持桌面端流式输出、原生审批、PTY、Worktree 和后台任务恢复。

## 进程边界

- 渲染进程：只负责显示和用户输入，开启 `sandbox` 与 `contextIsolation`，关闭 Node 集成。
- preload：暴露固定 IPC 方法，不暴露通用文件或 shell API。
- 主进程：管理密钥、配置、子进程和更新。
- OpenCode：只在用户选中的项目目录运行，通过临时配置接收供应商和研发安全规则。

## 后台任务

运行记录和权限请求只保存在 Electron 主进程内，渲染层通过受限 IPC 获取脱敏快照。有活动任务时关闭主窗口会转入系统托盘；无托盘环境降级为最小化。任务完成、失败或等待授权时可以发系统通知。再次启动应用由单实例锁唤醒原窗口，渲染层从 OpenCode 持久会话恢复消息，不复制执行任务。

执行服务由主进程按“项目 + 模型 + 权限配置”管理为服务池。不同项目可以并行执行，渲染层只订阅事件并按 `runId` 隔离当前视图；运行中的任务可留在后台，再切换项目继续工作。服务池默认最多保留五个进程，空闲且没有终端的旧服务会被回收，活动任务不会被回收。

集成终端默认直接运行安装包内置执行引擎的 `attach --mini`，通过本机回环 API 进入完整交互 TUI。主进程仅在 CLI 模式的 PTY 输出层统一终端标题和品牌显示为 WetoCode，不改变内部协议、配置目录或模型标识，也不要求用户额外安装 CLI。终端保留普通 Shell 模式；两种模式复用受控 PTY、项目授权、生命周期回收和固定 IPC 边界。

内置公共免费模型不携带发布者个人 API Key。未登录试用额度由第三方网关按公网 IP 统计，同一 NAT、公司或代理出口下的多个安装可能共享限额；用户添加的模型配置则使用各自系统密钥环中的凭据。

## Goal Loop

Goal 不是普通提示词，而是持久的会话目标契约。主进程保存目标、状态、轮次、Token、工具调用、预算和校验时间线。一轮 Agent 执行进入 idle 后，主进程创建只读子会话进行独立校验；只有结构化结论明确为完成时才结束，否则自动发起下一轮。校验失败按未完成处理，达到轮次、时间或 Token 上限时进入 `budget_limited`，保留状态供用户恢复。

## 项目文件与扩展

渲染进程不持有文件系统 API。文件列表、搜索、预览和引用通过受限 IPC 调用 OpenCode 文件服务，主进程再次验证项目授权与相对路径。Commands、Agents、MCP 与 LSP 使用 SDK 的真实发现结果；Skills 只扫描 OpenCode、Claude 和 Agent 兼容的标准目录。

## 本地统计

用量统计只写入本机聚合数据，不保存提示词、代码或模型输出。主进程按日和模型累计 Token、消息、工具调用、任务结果，原子写入并保留最近 400 天；渲染层只能读取 7 天、30 天或全部范围的汇总。

## 本地自动化

自动化计划持久化在 Electron 用户数据目录，支持单次、固定间隔、每天和每周本地时间。调度器只在应用进程运行时工作；存在启用计划时关闭窗口会继续驻留托盘。到期任务调用与手动任务相同的 Agent 入口，因此共享模型、权限、会话、用量统计、后台通知与结果恢复，不维护第二套执行循环。应用异常退出后，运行中的历史会标记为中断；过期的单次计划会在重启后补跑，周期计划则推进到下一次时间。

## 开发预览

预览进程由主进程管理，不向渲染层暴露通用 shell。命令必须来自受限的常见开发工具集合，不允许 shell 控制符或可执行文件路径；Windows 的 npm 类 `.cmd` 命令使用经过逐参数引用的 `cmd.exe /d /s /c` 启动。预览只接受本机回环 URL，捕获开发服务器输出并在应用退出时停止进程树。文件 IPC 会验证最深已存在父目录的真实路径，防止通过项目内符号链接访问项目外内容。

## 密钥

API Key 只以 Electron `safeStorage` 密文落盘。解密发生在启动 OpenCode 前，密钥通过子进程环境或临时内存配置传递。渲染层只能看到 `hasApiKey`。

自定义模型网关按 OpenAI Compatible、Anthropic Messages 或 Google Gemini 协议映射到对应的 AI SDK 适配器。连接测试由主进程直接执行最小生成请求，渲染层不接触已保存密钥；错误信息返回前会截断且不会回显密钥。

严格企业环境建议不在终端保存个人 Key，改由内网模型网关签发短期令牌。

## 长上下文

长上下文不是无限上下文。WetoCode 使用模型声明的真实窗口，并配置：

- 自动压缩历史
- 裁剪已经消费的工具输出
- 保留最近 24K tokens
- 预留 16K tokens 给模型输出

模型的实际窗口仍由供应商决定，界面不会承诺跨越物理上限。

## 自动更新

`electron-updater` 从 GitHub Releases 获取版本和签名元数据。生产发布必须满足：

- 替换仓库地址
- Windows Authenticode 签名
- macOS Developer ID 签名与 notarization
- Linux 包校验和
- CI 中生成并发布 `latest*.yml` 或对应元数据
- 企业环境允许切换到内部制品源并设置灰度通道
